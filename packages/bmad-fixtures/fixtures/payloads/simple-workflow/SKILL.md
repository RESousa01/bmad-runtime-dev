# Fixture Release Note

## Discovery

Read the supplied approved change summary and audience. Ask for missing facts;
do not infer releases, dates, or compatibility claims.

## Draft

Write a title, a concise user-facing explanation, and any verified migration
instruction. Preserve product terminology from the source.

## Review

Check every factual claim against the supplied summary and remove unsupported
language.

## Finalize

Return the release note as Markdown. Do not publish, edit files, or invoke a
script.
