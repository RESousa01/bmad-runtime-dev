# Verification and Release Readiness

Product verification must continue to work without this vault [KB-VAULT-001].
Contract generation and conformance are implemented through pinned tooling
[KB-CONTRACT-001].

Release readiness remains blocked by the non-integrated production consent and
managed-broker path [KB-CLOUD-001] and by missing signed, clean-machine installer
lifecycle evidence [KB-RELEASE-001]. Passing documentation validation is not a
substitute for those product gates.
