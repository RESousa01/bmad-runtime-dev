Plano Web: trabalhar com IA, com método e provas.

Esta apresentação cobre exclusivamente a aplicação Web (web_managed). Os projetos são importados ou carregados para uma área de trabalho gerida no Azure, onde a execução acontece remotamente. A aplicação Windows Desktop é um segundo produto, local-first, com autoridade local separada; os dois modelos partilham BMAD e Airlock, mas não misturam ficheiros, estado ou execução.

01 / Plano Web

Um caminho cloud seguro, do pedido ao resultado verificado.

No modelo Web, a IA sugere um plano, o Airlock verifica-o e um trabalhador Azure previamente aprovado executa apenas a ação aceite sobre uma cópia cloud gerida. A Sapphirus valida o resultado antes de considerar o trabalho concluído.

Área de trabalho

Uma pessoa explica o que precisa e fornece a informação relevante do projeto.

02

Método BMAD

O BMAD transforma o pedido em passos, responsabilidades e resultados claros.

03

Portal de IA

Uma ligação controlada escolhe a IA adequada e verifica o formato da resposta.

04

Verificação Airlock

A plataforma confirma o que vai acontecer, quem aprovou e quando a aprovação expira.

05

Trabalhador Azure

Um serviço cloud previamente aprovado realiza apenas a tarefa permitida, com limites fixos.

06

Prova e recuperação

O resultado é verificado, registado e fica disponível para revisão, repetição ou reversão.

03 / Como a IA é gerida

A IA ajuda no trabalho. Nunca controla a plataforma.

O Azure OpenAI ou o Microsoft Foundry liga-se através de um único portal controlado pela Sapphirus. A plataforma — e não o fornecedor de IA — guarda o histórico oficial do projeto e a informação de recuperação.

A Sapphirus guarda o histórico oficial

Conversas, pontos guardados, tentativas, provas e registos de recuperação permanecem sob controlo da aplicação.

As respostas importantes seguem um formato definido

A plataforma verifica duas vezes a resposta da IA para que informação inesperada ou em falta não entre silenciosamente no fluxo de trabalho.

As ferramentas de IA ficam desligadas por defeito

Uma IA não pode navegar, alterar ficheiros ou executar comandos sem passar pelo mesmo caminho de aprovação e segurança de qualquer ação.

Cada configuração de IA tem uma identidade

O modelo, localização, instruções, limites, definições de segurança, alternativa e resultados de testes ficam registados em conjunto.

Quatro perguntas antes de lançar

Segue o formato? O trabalho tem qualidade? É seguro e privado? A velocidade e o custo são aceitáveis?

Como uma configuração de IA chega à produção

CandidataRegistar o modelo e as definições que estão a ser consideradas.

06 / Desenvolvimento do plano Web

Não é necessário um servidor local potente.

Para construir e testar a aplicação Web, o computador de desenvolvimento precisa apenas das ferramentas fixadas no projeto. Os builds de trabalhadores, integrações Azure e tarefas isoladas acontecem em ambientes cloud controlados.

No computador de desenvolvimento

  • Usar versões fixas das ferramentas .NET, Node e Python
  • Verificar formatos de dados, regras de fluxo e exemplos repetíveis
  • Usar informação temporária de teste em vez de dados cloud reais
  • Usar o sealed_test_fake sem o apresentar como isolamento
  • Não exigir Docker, Kubernetes, emuladores cloud ou modelos de IA locais

No Azure do plano Web

  • Criar o software de trabalhador remotamente, sem exigir Docker no computador
  • Registar licenças, software incluído, verificações de segurança, origem, assinaturas e versões exatas
  • Usar um ambiente Azure de desenvolvimento separado para ligações reais de IA e dados
  • Executar a primeira ação genuína através de um Container Apps Job previamente aprovado
  • Proteger acessos e segredos, monitorizando cada ação e permitindo recuperação

Plano Web: uma área de trabalho cloud gerida, governada e recuperável.

Escolha esta opção quando a colaboração centralizada, a execução isolada no Azure e a governação organizacional forem mais importantes do que trabalhar diretamente sobre uma pasta local.
